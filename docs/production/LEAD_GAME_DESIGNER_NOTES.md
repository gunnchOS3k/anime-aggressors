# Lead Game Designer Notes

## Owns

- Combat rules and frame data targets
- Character roles and roster balance
- Move sets per fighter
- Beginner / intermediate / advanced depth
- Camera feel targets
- Game modes and win/loss loops

## Does not own

- Export scripts, CI, or Blender file structure (Engineer + Art)

## Deliverables

- Fighter specs in `docs/fighters/`
- Move choreography approval per fighter
- Playtest feedback triage

## Definition of fun (platform fighter)

- Readable silhouettes at default camera zoom
- Attacks telegraph with anticipation; impact sells with hitstop + VFX + camera
- Movement expresses fighter fantasy (pressure, spacing, rushdown)
